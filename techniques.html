Langages et Outils Utilisés

  1. Java : Utilisé pour implémenter la simulation. Java est un langage de programmation orienté objet largement utilisé dans de nombreux domaines, dont la simulation. En raison de sa portabilité et de sa robustesse, Java est souvent privilégié pour implémenter des simulations complexes, qu'elles soient utilisées dans les systèmes embarqués, les applications financières, ou même les simulations physiques et scientifiques.

    - Si vous souhaitez en sevoir plus sur Java, je vous invite a cliquer sur ce lien : Rechercher "programmation Java" sur Google

  2. Swing : Permet de créer l'interface graphique. Swing est non seulement utilisé pour créer des interfaces graphiques (GUI) robustes et multiplateformes, mais il est également très adapté pour des applications de simulation, en particulier pour la visualisation et l'interaction avec des modèles simulés. Lorsqu'il s'agit de visualiser des processus dynamiques ou de simuler des systèmes complexes, **Swing** permet de créer des interfaces qui permettent aux utilisateurs d'interagir avec la simulation en temps réel.

    Pour apprendre Java Swing, consultez cette ressource utile : Java Swing Tutorial - Oracle

  3. HTML/CSS : Utilisés pour le développement du site web accompagnant le projet.
    • HTML (HyperText Markup Language) est utilisé pour structurer le contenu de la page web. Il définit les éléments essentiels comme les titres, les paragraphes, les liens, les images, les formulaires, et plus encore.
    • CSS (Cascading Style Sheets) est utilisé pour définir l'apparence de ces éléments. Il permet de styliser les pages avec des couleurs, des polices, des mises en page, des animations, etc.

    - Pour apprendre HTML et CSS, consultez cette ressource utile : W3Schools - Learn HTML, CSS, JavaScript

Explication de l'Interface

L'interface utilisateur offre les fonctionnalités suivantes :

  • La possibilité de choisir la taille de la grille pour la simulation.
  • La configuration du nombre de bêtes présentes dans l'environnement.
  • L'ajout ou la suppression d'obstacles qui influencent le comportement des bêtes.

Gestion des Performances

Pour assurer des performances optimales, plusieurs optimisations ont été réalisées :

  • Utilisation d'une gestion efficace des threads pour exécuter les actions des bêtes en parallèle.
  • Réduction du nombre d'objets actifs en mémoire pour limiter la consommation de ressources.
  • Implémentation d'une mise à jour différée pour diminuer les calculs en temps réel.
